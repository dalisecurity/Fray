CLI Redesign: 61 → 20 Visible Commands IMPLEMENTED

Status: Implemented in commit ac1b1a8 on hugo branch. All 61 old commands still work (backward compat) — deprecated ones print migration hints. fray --help now shows 20 grouped commands instead of 61 flat ones.

Design Principles

1. Progressive disclosure — Show 20 commands, not 61
2. Verb-noun pattern — fray report company, not fray company-report
3. One obvious starting point — fray go or just fray <url>
4. Group by intent — Users think "I want to test", not "I need the agent module"
5. Bare URL shortcut — fray example.com = fray go example.com

Quick Start Examples

# First time — guided wizard
fray

# One-command full assessment
fray go https://example.com

# Deep stealth recon
fray recon https://example.com --deep --stealth

# XSS test with adaptive evolution
fray test https://example.com -c xss --smart --mutate

# Auto-scan with browser and blind detection
fray scan https://example.com -c xss --browser --blind

# Self-improving agent
fray test https://example.com --agent -c xss --rounds 5

# Authenticated testing
fray auth session login https://example.com/login --name mysite --cookie "sid=abc"
fray test https://example.com -c sqli --load-session mysite

# CI/CD pipeline
fray go https://staging.example.com --ci --fail-on high --sarif

# Batch enterprise
fray recon --batch domains.txt -w 8 --profile deep -o results.jsonl
fray report posture results.jsonl -o posture_report.md

# Continuous monitoring
fray monitor example.com --interval 6h --webhook https://hooks.slack.com/...

# Threat intelligence
fray intel feed --since 7d --auto-add --test-target https://example.com

# Create a plugin
fray plugin init --name slack_notifier
# edit ~/.fray/plugins/slack_notifier.py, auto-loads next run
